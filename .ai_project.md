# 光核超级服务 Project Memory

## Purpose

Independent Tauri 2 desktop rebuild of the Codex usage dashboard. The user-visible product name is 光核超级服务; compatible internal technical identifiers remain `paishu-agi`, `paishu_agi`, and `com.paishu.agi`.

## Stack

- Frontend: React, TypeScript, Vite, Vitest, Testing Library
- Desktop: Tauri 2, Rust
- Data: Codex app-server JSON-RPC, local `.codex` SQLite/session JSONL, localhost PAISHU PostgreSQL + pgvector knowledge service

## Current Product Rules

- UI is Simplified Chinese first.
- First screen is the usable desktop dashboard, not a landing page.
- macOS development mode is the current target; unsigned/ad-hoc signed macOS arm64 test packaging is available for local tester distribution.
- Current macOS DMG output for this rebrand is `光核超级服务_1.5.0_aarch64.dmg` in the source root; Tauri also writes the canonical bundle under `src-tauri/target/release/bundle/dmg/`.
- The Windows NSIS installer should use the 光核超级服务 product name on its next Windows build; internal package identifiers remain unchanged for compatibility.
- Canonical public source and installer repository: `https://github.com/PAISHU-AI/codex-PAISHU-source`; publish the verified root-level DMG together with source updates on `main`.
- Official native mode reads Codex official/default account state and does not need API endpoint, model, reasoning, or speed controls.
- API relay mode stores endpoint/model/reasoning/speed settings and synchronizes them to the user Codex `config.toml` when saving settings. It also accepts a one-time API Key for Codex `auth.json`; the key is written to `OPENAI_API_KEY` with `auth_mode = "apikey"` and is not stored in the app settings file. Endpoint input accepts a base API address and normalizes to exactly one `/v1`; dashboard statistics use local SQLite/JSONL data in this mode.
- Official native mode save restores the Codex config to official ChatGPT auth shape by editing the current config in place, preserving unrelated sections such as projects, trust records, and MCP servers. It also sets Codex `auth.json` to `auth_mode = "chatgpt"` and clears `OPENAI_API_KEY`.
- Official native mode dashboard statistics use official app-server account data, including daily usage buckets returned by `account/usage/read`.
- Current macOS installs may bundle the Codex CLI inside ChatGPT at `/Applications/ChatGPT.app/Contents/Resources/codex`; this path is a first-class detection candidate, and `app-server` is invoked as its `codex app-server` subcommand.
- Official quota windows are classified by the returned `windowDurationMins`, rather than the app-server `primary` / `secondary` field names. Current ChatGPT-bundled Codex can return only the 10,080-minute 7-day window as `primary`; the dashboard displays only the official 7-day remaining percentage in a single-ring glass status card, with used percent and reset time, and does not present a retired 5-hour quota or local substitute.
- Official-login display has a sanitized local fallback: `auth.json` is read only for `auth_mode` and the presence of a ChatGPT access token, never returned to the frontend. This keeps login state visible if an ephemeral app-server request times out.
- Member value progress uses the current billing cycle anchored to the configured original membership open date, not all-time usage since opening.
- Refresh should not mix primary data sources: official mode uses official account usage, API mode uses local usage.
- The dashboard replaces the old visible task board with an isolated Skills board. It scans local Codex skills, shows only skill names/descriptions, and keeps delete/disable operations restricted to user skills via safe archive/disabled folders.
- Skills board supports status filtering for all/enabled/disabled skills. Disabled skills are read from `~/.codex/skills-disabled`, shown in the disabled filter, and can be enabled from the board by moving them back to `~/.codex/skills`.
- Skills board can translate the selected skill description to Simplified Chinese through Google Translate and toggle back to the original text without mutating skill files.
- The full-width Knowledge Board connects to the localhost PAISHU vector knowledge service through `src-tauri/src/knowledge_board.rs`. The Rust boundary reads the local service token without exposing it to React; the board shows database capacity, documents, chunks, read latency, success/failure counts, filters, details, and reversible enable/disable actions.
- Knowledge Board refresh auto-syncs governed local knowledge packages before reading the dashboard. The Rust boundary detects `PAISHU_KNOWLEDGE_RETRIEVAL_DIR`, `PAISHU_KNOWLEDGE_RETRIEVAL_DIRS`, the development `knowledge-retrieval/` folder, and the default `~/Desktop/GUANGHE-PAISHU/knowledge-retrieval`; it reads each package `metadata.yml`, skips draft/deprecated/archived/quarantined packages or `ingestion.enabled: false`, syncs only `kb/` when `ingestion.mode: kb_only`, and carries package `status`, `access_tier`, and `owner` into the ingest command.
- The knowledge detail pane reads a bounded overview from the active persisted document revision and can temporarily translate it to Simplified Chinese without mutating knowledge content.
- The macOS shell is fully opaque. The title-bar minimize button sends the window to Dock, closing hides it to the top menu bar, the tray menu exposes both paths, and a Dock reopen restores/focuses the main window.
- Brand assets are source-controlled: `src-tauri/icons/source-icon.png` contains the current 光核超级服务 desktop icon source and generates all platform icons; `src/assets/paishu-agi-logo.png` contains the current in-app logo while retaining its compatibility filename. README screenshots live under `docs/assets/` for quota/dashboard, knowledge, Skills, and API relay views.
